Cấu hình CORS (Cực kỳ quan trọng)\nNếu không có cấu hình này, trình duyệt sẽ chặn mọi request từ Frontend. Vui lòng thêm vào cấu hình Spring Security hoặc WebConfig:\n\n```java\n@Configuration\npublic class WebConfig implements WebMvcConfigurer {\n    @Override\n    public void addCorsMappings(CorsRegistry registry) {\n        registry.addMapping(\"/api/v1/**\")\n                .allowedOrigins(\"http://localhost:3000\")\n                .allowedMethods(\"GET\", \"POST\", \"PUT\", \"DELETE\", \"OPTIONS\")\n                .allowedHeaders(\"Authorization\", \"Content-Type\", \"X-Internal-Secret\")\n                .allowCredentials(true);\n    }\n}\n```\n\n### 2. Định dạng Response chuẩn\nFrontend sử dụng Axios Interceptor để bóc tách dữ liệu. Mọi API thành công cần trả về cấu trúc:\n```json\n{\n  \"status\": \"success\",\n  \"message\": \"Thông báo thành công\",\n  \"data\": { ... object hoặc array dữ liệu ... }\n}\n```\n\n### 3. Quy tắc đặt tên trường (Field Names)\nĐể đồng bộ với UI, Backend cần trả về đúng tên trường như sau:\n- **Wallet**: Dùng `balance` (thay vì `walletBalance`).\n- **Order**: Dùng `totalPrice`, `status`, `fishCount`.\n- **User**: Dùng `fullName`, `email`, `role`, `status`.\n\n### 4. Cơ chế JWT & Refresh Token\nFrontend đã cài đặt cơ chế **Silent Refresh**. Quy trình như sau:\n1. Khi Login: Trả về `accessToken` và `refreshToken`.\n2. Khi gọi API: Frontend đính kèm `Authorization: Bearer <accessToken>`.\n3. Khi lỗi 401 (AccessToken hết hạn): Frontend tự động gọi `POST /auth/refresh` kèm `{ \"refreshToken\": \"...\" }` để đổi lấy Token mới.\n\n### 5. Logic AI Webhook (Price Adjustment)\nKhi dịch vụ AI gửi kết quả về:\n- Nếu `ai_fishCount < ordered_quantity`: Backend cần tự động tính lại `totalPrice` và tạo một bản ghi `REFUND` vào bảng `Transaction`.\n- Tiền hoàn lại sẽ cộng vào `balance` của người mua.\n\n### 6. Endpoint Admin\n- Đảm bảo các route `/admin/**` chỉ cho phép người dùng có `ROLE_ADMIN` truy cập.\n- Phí sàn cố định là **5%** (`totalPrice * 0.05`).\n\n---\n💡 **Mẹo Debug:** Nếu Frontend không hiển thị dữ liệu, hãy mở Console (F12) để xem lỗi Network. Nếu là `CORS error`, hãy kiểm tra lại mục số 1 ở trên."
